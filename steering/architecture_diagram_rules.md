# AWS Architecture Diagram Guidelines

Create an AWS architecture diagram using the awslabs aws_diagram_mcp_server with the following specifications:

## Technical Requirements
- Use the latest AWS service icons and logos
- Set diagram direction from left to right (direction="LR")
- Focus on core services and components only
- Exclude: CloudWatch, Security Groups, X-Ray tracing, and other monitoring/security overlay services

## Architecture Flow
- Start with user/client on the left side
- Show data flow moving from left to right through the architecture
- End with data storage or processing services on the right side

## Diagram Structure
- Group related services using Clusters where logical
- Show clear connections between services with arrows
- Use descriptive labels for each service instance
- Maintain clean, readable layout

Please generate the diagram code focusing on the main architectural components and their relationships.
