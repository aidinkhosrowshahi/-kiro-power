# AWS Development Best Practices Power

A comprehensive Kiro power that provides AWS development standards, best practices, and guidelines for building secure, reliable, and performant applications.

## Installation

Install this power in Kiro by importing from GitHub or through the Kiro Powers interface.

## What's Included

This power includes curated steering files covering:

- **AWS Well-Architected Framework** - Security, reliability, and performance principles
- **Infrastructure as Code** - Terraform, CloudFormation, and SAM best practices
- **Application Development** - Python 3.13 and React standards
- **Agentic Applications** - Strands Agents and AgentCore guidelines
- **Deployment Patterns** - CloudFront/S3 hosting templates
- **Development Workflow** - Git commit standards and automation

## Steering Files

All steering files are automatically included in your context:

- `well-architected-context.md` - AWS Well-Architected Framework principles
- `terraform_best_practices.md` - Terraform development standards
- `python_best_practices.md` - Python 3.13 best practices
- `react_best_practices.md` - React development guidelines
- `automation-notes.md` - IaC and scripting standards
- `architecture_diagram_rules.md` - AWS diagram creation guidelines
- `approved-model-for-building-agentic.md` - Approved models for agentic apps
- `agentcore_strands_requirements.md` - Required modules for Strands/AgentCore
- `CloudFront_S3_setup_for_UI_app_website.md` - Website hosting templates
- `git_commit_rules.md` - Git commit message standards
- `minimal-engineering-approach.md` - Minimal engineering principles
- `aws_cli_usage.md` - AWS CLI authentication guidelines

## Use Cases

- Setting up new AWS projects with proper standards
- Reviewing and improving existing infrastructure
- Building agentic applications with Strands and AgentCore
- Deploying serverless applications
- Creating secure website hosting
- Following AWS best practices

## Requirements

- AWS CLI configured
- Python 3.13 (for backend development)
- Node.js/npm (for React development)
- Terraform or AWS SAM CLI (for infrastructure)

## Security Note

This power contains no sensitive information, credentials, or account-specific data. All templates use CloudFormation variables for account IDs and resource names.
